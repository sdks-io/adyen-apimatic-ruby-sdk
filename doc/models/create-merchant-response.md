
# Create Merchant Response

## Structure

`CreateMerchantResponse`

## Fields

| Name | Type | Tags | Description |
|  --- | --- | --- | --- |
| `business_line_id` | `String` | Optional | The unique identifier of the [business line](https://docs.adyen.com/api-explorer/#/legalentity/latest/post/businessLines). |
| `company_id` | `String` | Optional | The unique identifier of the company account. |
| `description` | `String` | Optional | Your description for the merchant account, maximum 300 characters.<br><br>**Constraints**: *Maximum Length*: `300` |
| `id` | `String` | Optional | The unique identifier of the merchant account. If Adyen set up a template for the `reference`, then the `id` will have the same value as the `reference` that you sent in the request. Otherwise, the value is generated by Adyen. |
| `legal_entity_id` | `String` | Optional | The unique identifier of the [legal entity](https://docs.adyen.com/api-explorer/#/legalentity/latest/post/legalEntities). |
| `pricing_plan` | `String` | Optional | Partner pricing plan for the merchant, applicable for merchants under AfP managed company accounts. |
| `reference` | `String` | Optional | Your reference for the merchant account. |
